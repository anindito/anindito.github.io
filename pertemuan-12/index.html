Struct sebagai Parameter: Pass by Value

struct Point {
    int x, y;
};

void tampilkanPoint(Point p) {
    cout << "(" << p.x << ", " << p.y << ")" << endl;
}

void geserPoint(Point p, int dx, int dy) {
    p.x += dx;
    p.y += dy;
    // Perubahan tidak mempengaruhi struct asli
}

int main() {
    Point p1 = {10, 20};
    tampilkanPoint(p1);  // (10, 20)
    geserPoint(p1, 5, 5);
    tampilkanPoint(p1);  // Masih (10, 20)
}
Pass by value menyalin seluruh struct ‚Üí Overhead untuk struct besar

Struct sebagai Parameter: Pass by Reference

struct Mahasiswa {
    string nama;
    float ipk;
};

// Pass by reference untuk modifikasi
void updateIPK(Mahasiswa &mhs, float ipkBaru) {
    mhs.ipk = ipkBaru;
}

// Const reference untuk efisiensi tanpa modifikasi
void tampilkan(const Mahasiswa &mhs) {
    cout << mhs.nama << ": " << mhs.ipk << endl;
}

int main() {
    Mahasiswa m1 = {"Alice", 3.50};
    tampilkan(m1);    // Alice: 3.50
    updateIPK(m1, 3.75);
    tampilkan(m1);    // Alice: 3.75
}

Fungsi yang Mengembalikan Struct

struct Point {
    double x, y;
};

Point buatPoint(double x, double y) {
    Point p;
    p.x = x;
    p.y = y;
    return p;
}

Point titikTengah(Point p1, Point p2) {
    Point tengah;
    tengah.x = (p1.x + p2.x) / 2.0;
    tengah.y = (p1.y + p2.y) / 2.0;
    return tengah;
}

int main() {
    Point p1 = buatPoint(0, 0);
    Point p2 = buatPoint(10, 10);
    Point tengah = titikTengah(p1, p2);  // (5, 5)
}

Union: Berbagi Memori

Union mirip struct, tetapi semua member berbagi lokasi memori yang sama 
union Data {
    int intValue;
    float floatValue;
    char charValue;
};

int main() {
    Data d;
    d.intValue = 42;
    cout << d.intValue << endl;     // 42
    
    d.floatValue = 3.14;
    cout << d.floatValue << endl;   // 3.14
    cout << d.intValue << endl;     // ??? (tidak valid lagi!)
}
Hanya satu member yang valid pada satu waktu!

Struct vs Union: Penggunaan Memori

Perbandingan Struct vs Union

Aspek Struct Union
Memori Setiap member punya ruang sendiri Semua member berbagi ruang yang sama
Ukuran Jumlah ukuran semua member Ukuran member terbesar
Akses Semua member bisa diakses bersamaan Hanya satu member valid pada satu waktu
Kegunaan Representasi objek dengan banyak atribut Menghemat memori untuk data yang mutually exclusive
Risiko Lebih banyak menggunakan memori Mudah terjadi bug jika tidak hati-hati

Tagged Union

Kombinasi enum + union untuk melacak member yang aktif 
enum TipeData { TIPE_INT, TIPE_FLOAT, TIPE_STRING };

struct TaggedUnion {
    TipeData tipe;  // Tag
    union {
        int intValue;
        float floatValue;
        char stringValue[50];
    } data;
};

void tampilkan(const TaggedUnion &tu) {
    switch(tu.tipe) {
        case TIPE_INT:
            cout << "Integer: " << tu.data.intValue << endl;
            break;
        case TIPE_FLOAT:
            cout << "Float: " << tu.data.floatValue << endl;
            break;
        case TIPE_STRING:
            cout << "String: " << tu.data.stringValue << endl;
            break;
    }
}

Typedef: Alias Tipe Data

typedef membuat nama alternatif untuk tipe data yang sudah ada 
// Typedef untuk tipe primitif
typedef unsigned long long ull;
typedef long double ldouble;

// Typedef untuk struct
struct Mahasiswa {
    string nama;
    float ipk;
};
typedef struct Mahasiswa Mhs;

int main() {
    ull angkaBesar = 18446744073709551615ULL;
    ldouble pi = 3.14159265358979323846L;
    
    Mhs m1 = {"Alice", 3.85};
}

Using Declaration (C++11)

Cara modern untuk membuat alias tipe (lebih konsisten dan readable) 
// Using untuk tipe primitif
using Integer = int;
using Real = double;

// Using untuk tipe kompleks
using StringVector = vector;
using IntPtr = int*;

// Using untuk struct
struct Point { double x, y; };
using Point2D = Point;

int main() {
    Integer angka = 42;
    Real desimal = 3.14159;
    StringVector nama = {"Alice", "Bob"};
    Point2D p = {10.5, 20.3};
}

Typedef vs Using

Aspek typedef using
Sintaks typedef old_type new_name; using new_name = old_type;
Readability Kurang intuitif untuk tipe kompleks Lebih jelas: nama = tipe
Template Tidak support template aliases Support template aliases
Rekomendasi Legacy code ‚úÖ Gunakan ini untuk kode baru (C++11+)
Best Practice: Gunakan using untuk kode C++11 dan setelahnya

Enum (Enumeration)

Enum adalah tipe data yang terdiri dari sekumpulan konstanta bernama 
enum Hari {
    SENIN,    // 0
    SELASA,   // 1
    RABU,     // 2
    KAMIS,    // 3
    JUMAT,    // 4
    SABTU,    // 5
    MINGGU    // 6
};

enum Status {
    PENDING = 1,
    PROCESSING = 2,
    COMPLETED = 3,
    CANCELLED = 0
};

int main() {
    Hari hariIni = RABU;
    Status status = PROCESSING;
    
    cout << hariIni << endl;   // 2
    cout << status << endl;     // 2
}

Enum Class (Scoped Enum)

Enum class memberikan type safety dan mencegah name pollution 
enum class Warna {
    MERAH,
    HIJAU,
    BIRU,
    KUNING
};

enum class Priority {
    LOW = 1,
    MEDIUM = 5,
    HIGH = 10
};

int main() {
    Warna w = Warna::BIRU;  // Harus pakai scope
    Priority p = Priority::HIGH;
    
    // cout << w << endl;  // ERROR! Tidak auto-convert ke int
    cout << static_cast(p) << endl;  // OK: 10
}

Enum vs Enum Class

Aspek enum enum class
Scope Global (unscoped) Scoped (harus pakai ::)
Konversi ke int Otomatis Harus explicit cast
Type safety Lemah Kuat
Name collision Bisa terjadi Tidak akan terjadi
Rekomendasi Legacy code ‚úÖ Gunakan ini (C++11+)

Aplikasi: Sistem Inventory

enum class Kategori { ELEKTRONIK, MAKANAN, PAKAIAN };

struct Barang {
    string kode;
    string nama;
    Kategori kategori;
    int stok;
    float harga;
};

void tambahStok(Barang &b, int jumlah) {
    b.stok += jumlah;
}

bool kurangiStok(Barang &b, int jumlah) {
    if (b.stok >= jumlah) {
        b.stok -= jumlah;
        return true;
    }
    return false;
}

float hitungNilaiInventory(Barang inventory[], int n) {
    float total = 0;
    for (int i = 0; i < n; i++) {
        total += inventory[i].stok * inventory[i].harga;
    }
    return total;
}

üí° Best Practices

1. Gunakan struct untuk mengelompokkan data yang berhubungan
struct Point { double x, y; };
2. Pass by const reference untuk struct besar
void display(const LargeStruct &s);
3. Gunakan enum class untuk type safety (C++11+)
enum class Status { ACTIVE, INACTIVE };
4. Gunakan tagged union jika perlu union
Tambahkan enum untuk track member yang aktif
5. Gunakan using daripada typedef (C++11+)
Lebih readable dan konsisten

‚ö†Ô∏è Common Pitfalls

1. Lupa titik koma setelah definisi struct 
struct Point {
    int x, y;
}; // ‚Üê HARUS ADA!
2. Mengakses union member yang tidak aktif 
union Data { int i; float f; };
Data d;
d.i = 10;
cout << d.f; // WRONG! d.f tidak valid
3. Name pollution dengan enum biasa 
enum Color { RED, GREEN };
enum Signal { RED, GREEN }; // ERROR: redefinition!
// Gunakan enum class!

Struct vs Class (Preview)

Dalam C++, struct sebenarnya adalah class dengan perbedaan kecil:
Aspek struct class
Default Access public private
Inheritance Default public private
Kegunaan Umum Data grouping sederhana OOP dengan encapsulation
Struct adalah jembatan menuju OOP! üåâ

Kapan Menggunakan Apa?

Gunakan Ketika Contoh
Struct Mengelompokkan data berhubungan Point, Rectangle, Student, Book
Union Data mutually exclusive, hemat memori Tagged union untuk variant types
Typedef/Using Membuat kode lebih readable StudentID, Matrix2D, Callback
Enum Set nilai terbatas (legacy) Legacy code compatibility
Enum Class Set nilai terbatas + type safety Status, Color, Direction, Gender

üìù Ringkasan

Struct: Mengelompokkan data berbeda menjadi satu tipe

Union: Member berbagi memori (hemat, tapi hati-hati)

Typedef/Using: Membuat alias untuk tipe data

Enum: Konstanta bernama (gunakan enum class)

Array of Struct: Mengelola koleksi data terstruktur

Nested Struct: Struktur data hierarkis

Key Takeaway: Tipe data bentukan membantu mengorganisasi data kompleks dengan lebih baik dan menjadi fondasi untuk OOP!
