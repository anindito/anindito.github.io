Itulah Network Sniffing!

Apa itu Network Sniffing?

Definisi:

Network sniffing adalah proses menangkap dan menganalisis data yang lewat di jaringan komputer.

Penggunaan Legal:

  • Troubleshooting jaringan
  • Monitoring keamanan
  • Analisis performa

Penggunaan Illegal:

  • Mencuri password
  • Membaca email orang lain
  • Mengintip data pribadi

üåê Konsep Dasar

Bagaimana Data Bergerak di Jaringan?

Cara Kerja Komunikasi Jaringan

Saat Anda browsing atau kirim pesan, data tidak langsung ke tujuan.

Data dipecah jadi paket-paket kecil:

üì¶ Paket 1: "Halo, " üì¶ Paket 2: "ini " üì¶ Paket 3: "saya!"

Setiap paket punya "alamat":

  • IP Address: Seperti alamat rumah (contoh: 192.168.1.10)
  • MAC Address: Seperti nomor KTP kartu jaringan (contoh: AA:BB:CC:DD:EE:FF)

Perbedaan IP vs MAC Address

Aspek IP Address MAC Address
Fungsi Alamat logis dalam jaringan Alamat fisik kartu jaringan
Sifat Bisa berubah Tetap (unique per device)
Contoh 192.168.1.100 00:1A:2B:3C:4D:5E
Layer Layer 3 (Network) Layer 2 (Data Link)

Apa itu ARP?

ARP = Address Resolution Protocol

Protokol untuk menterjemahkan IP Address ke MAC Address

Analogi:

Seperti buku telepon yang menghubungkan nama (IP) dengan nomor telepon (MAC)

Kenapa penting?

Komputer perlu tahu MAC address untuk mengirim data secara fisik di jaringan lokal

Cara Kerja ARP (Normal)

1 Komputer A mau kirim data ke 192.168.1.10

2 Komputer A broadcast: "Siapa yang punya IP 192.168.1.10?"

3 Komputer B jawab: "Saya! MAC saya adalah AA:BB:CC:DD:EE:FF"

4 Komputer A simpan info ini di ARP cache

5 Komputer A kirim data langsung ke MAC address Komputer B

ARP Cache Table

Setiap komputer punya tabel ARP yang menyimpan mapping IP-to-MAC:

IP Address        MAC Address           Type
192.168.1.1      00:1A:2B:3C:4D:5E    dynamic
192.168.1.10     AA:BB:CC:DD:EE:FF    dynamic
192.168.1.20     11:22:33:44:55:66    dynamic

Masalah: ARP tidak punya mekanisme autentikasi! Semua komputer percaya jawaban ARP tanpa verifikasi!

Serangan yang Membahayakan

Apa itu ARP Poisoning?

Definisi:

ARP Poisoning (ARP Spoofing) adalah teknik di mana penyerang mengirim ARP reply palsu untuk mengubah ARP cache korban

Tujuan:

  • Mengalihkan traffic jaringan ke komputer penyerang
  • Menjadi "man in the middle"
  • Membaca/memodifikasi data yang lewat

Analogi: Penipu Paket

Situasi Normal:

üì¨ Anda ‚û°Ô∏è Kurir Jujur ‚û°Ô∏è üè† Teman

ARP Poisoning:

üë§ Penipu berpura-pura jadi kurir:

"Hai, saya kurir! Kasih paket ke saya!"

üì¨ Anda ‚û°Ô∏è üë§ Penipu (buka paket) ‚û°Ô∏è üè† Teman

Teman Anda tetap terima paket, tapi penipu sudah tahu isinya!

Cara Kerja ARP Poisoning

Sebelum Attack:

Client: "Gateway IP 192.168.1.1 = MAC: AA:AA:AA"

Gateway: "Client IP 192.168.1.10 = MAC: BB:BB:BB"

Saat Attack:

Attacker kirim ARP palsu ke Client:

"Gateway 192.168.1.1 sekarang MAC: CC:CC:CC (MAC Attacker)"

Attacker kirim ARP palsu ke Gateway:

"Client 192.168.1.10 sekarang MAC: CC:CC:CC (MAC Attacker)"

Hasil:

Semua traffic Client ‚ÜîÔ∏è Gateway lewat Attacker!

Langkah-Langkah ARP Poisoning

1 Reconnaissance

Scan jaringan, cari target (korban dan gateway)

2 Enable IP Forwarding

Agar data terus mengalir (tidak putus koneksi korban)

3 Kirim ARP Palsu

Broadcast ARP reply palsu secara terus-menerus

4 Start Sniffing

Tangkap dan analisis semua traffic yang lewat

5 Extract Data

Cari password, cookies, data sensitif

üëπ MITM Attack

Man-in-the-Middle

Apa itu MITM Attack?

Definisi:

Serangan di mana penyerang diam-diam menjadi perantara antara dua pihak yang berkomunikasi

Penyerang bisa:

  • Membaca semua komunikasi
  • Memodifikasi data yang dikirim
  • Menyuntikkan data palsu
  • Mencuri kredensial

Analogi MITM: Penyadap Telepon

Komunikasi Normal:

üë® Andi telepon üë© Budi

"Andi: Hai Budi!" ‚û°Ô∏è "Budi: Hai Andi!"

MITM Attack:

üë® Andi ‚û°Ô∏è üòà Charlie (penyadap) ‚û°Ô∏è üë© Budi

Andi: "Hai Budi!"

Charlie dengar, teruskan: "Hai Budi!"

Budi: "Hai Andi!"

Charlie dengar, teruskan: "Hai Andi!"

Andi dan Budi tidak tahu Charlie mendengarkan!

Tipe-Tipe MITM Attack

1. ARP Spoofing: Mengubah ARP cache (yang kita bahas tadi)

2. DNS Spoofing: Mengalihkan domain ke IP palsu

3. HTTPS Spoofing: Fake certificate untuk intercept HTTPS

4. WiFi Eavesdropping: Membuat fake WiFi hotspot

5. Session Hijacking: Mencuri session cookie

Skenario Real: Evil Twin Attack

Setup:

üìç Di coffee shop ada WiFi: "CoffeeShop_Free"

Attack:

üòà Hacker buat WiFi palsu dengan nama sama: "CoffeeShop_Free"

üí™ Signal lebih kuat dari yang asli!

Result:

  • Korban connect ke WiFi palsu
  • Semua traffic lewat laptop hacker
  • Password, email, banking data dicuri!

SSL Stripping Attack

Teknik untuk "menurunkan" koneksi HTTPS jadi HTTP

Cara Kerja:

1 Korban minta akses ke https://bank.com

2 Request lewat attacker (MITM)

3 Attacker forward ke server asli (HTTPS)

4 Attacker terima response HTTPS

5 Attacker ubah semua link jadi HTTP

6 Korban lihat http://bank.com (tidak aman!)

Dampak MITM Attack

üí≥ Finansial: Pencurian data kartu kredit, internet banking

üîê Privasi: Email, pesan pribadi, foto terbaca

üè¢ Bisnis: Data perusahaan, rahasia dagang bocor

üé≠ Identitas: Account takeover, identity theft

Cara Deteksi dan Melindungi Diri

Cara Deteksi ARP Poisoning

1. Cek ARP Cache secara manual

# Windows
arp -a

# Linux/Mac
arp -n

Cari duplikat MAC address untuk IP berbeda!

2. Gunakan tools monitoring

  • XArp (Windows) - alert saat ada ARP changes
  • arpwatch (Linux) - monitor ARP table

Tanda-Tanda Sedang Kena MITM

  • ‚ö†Ô∏è Koneksi tiba-tiba jadi lambat
  • ‚ö†Ô∏è Browser warning tentang certificate
  • ‚ö†Ô∏è Website tiba-tiba jadi HTTP (bukan HTTPS)
  • ‚ö†Ô∏è Session sering logout sendiri
  • ‚ö†Ô∏è Aktivitas aneh di account
  • ‚ö†Ô∏è WiFi dengan nama sama tapi signal beda

üõ°Ô∏è Cara Melindungi Diri

#1 PALING PENTING: Gunakan VPN!

VPN mengenkripsi semua traffic Anda

#2: Selalu cek HTTPS

Jangan login di website tanpa üîí icon

#3: Aktifkan 2FA (Two-Factor Authentication)

Meskipun password dicuri, akun tetap aman

Best Practices di WiFi Publik

  • WAJIB pakai VPN
  • Jangan akses banking/data sensitif
  • Disable automatic WiFi connection
  • Verifikasi nama WiFi ke staff
  • Forget network setelah selesai
  • Enable firewall
  • Update OS dan aplikasi

Mitigasi untuk Administrator

1. Enable Dynamic ARP Inspection (DAI)

Validasi ARP packets di switch

2. Implement Port Security

Batasi MAC address per port

3. Client Isolation di WiFi

Prevent client-to-client communication

4. Network Monitoring

Deploy IDS/IPS untuk deteksi anomali

Konfigurasi Static ARP

Untuk device kritikal, bisa set static ARP entry:

# Windows
arp -s 192.168.1.1 00-aa-bb-cc-dd-ee

# Linux
sudo arp -s 192.168.1.1 00:aa:bb:cc:dd:ee

# Permanent di Linux
sudo nano /etc/ethers
# Add: 192.168.1.1 00:aa:bb:cc:dd:ee

Note: Hanya praktis untuk jaringan kecil
