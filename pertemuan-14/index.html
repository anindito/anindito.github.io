Pengenalan Pemrograman Berorientasi Objek (OOP)

Dasar-Dasar Pemrograman

Tujuan Pembelajaran

Setelah pertemuan ini, mahasiswa diharapkan dapat:

  • Memahami paradigma pemrograman berorientasi objek
  • Memahami konsep class dan object serta perbedaannya
  • Membuat dan menggunakan class dengan constructor dan destructor
  • Menerapkan enkapsulasi dengan access specifier
  • Memahami dan mengimplementasikan inheritance
  • Memahami konsep dasar polymorphism dengan virtual functions
  • Mengembangkan aplikasi sederhana menggunakan prinsip OOP

Paradigma Pemrograman

Prosedural

  • Data terpisah dari fungsi
  • Fokus pada algoritma
  • Program = urutan instruksi
  • Contoh: C, Pascal

Berorientasi Objek

  • Data + fungsi dalam objek
  • Fokus pada objek & interaksi
  • Program = kumpulan objek
  • Contoh: C++, Java, Python

Empat Pilar OOP

1. Encapsulation

Penggabungan data dan method + menyembunyikan detail internal

2. Inheritance

Pewarisan atribut dan method dari class lain

3. Polymorphism

Satu interface untuk berbagai tipe objek

4. Abstraction

Menyembunyikan kompleksitas, menampilkan yang penting

Class = Blueprint atau template

Constructor

Constructor adalah method khusus yang dipanggil otomatis saat object dibuat

  • Nama sama dengan nama class
  • Tidak memiliki return type
  • Digunakan untuk inisialisasi

Jenis Constructor


class Mahasiswa {
private:
    string nama;
    float ipk;

public:
    // Default constructor
    Mahasiswa() {
        nama = "Belum diisi";
        ipk = 0.0;
    }
    
    // Parameterized constructor
    Mahasiswa(string n, float i) {
        nama = n;
        ipk = i;
    }
};

Constructor Initializer List


class Mahasiswa {
private:
    string nama;
    string nim;
    float ipk;

public:
    // Constructor dengan initializer list
    Mahasiswa(string n, string ni, float i) 
        : nama(n), nim(ni), ipk(i) {
        cout << "Mahasiswa " << nama << " dibuat" << endl;
    }
    
    // Lebih efisien!
};

Destructor

Destructor adalah method khusus yang dipanggil otomatis saat object dihancurkan

  • Nama sama dengan class, diawali tilde (~)
  • Tidak memiliki return type dan parameter
  • Digunakan untuk cleanup

~Mahasiswa() {
    cout << "Object " << nama << " dihancurkan" << endl;
}

Encapsulation (Enkapsulasi)

Data Hiding: Membuat data private dan mengaksesnya melalui public methods

Keuntungan:

  • Melindungi integritas data
  • Validasi saat mengubah data
  • Mudah dimodifikasi tanpa mengubah interface
  • Meningkatkan maintainability

Getter dan Setter Methods


class Mahasiswa {
private:
    float ipk;

public:
    // Getter
    float getIPK() {
        return ipk;
    }
    
    // Setter dengan validasi
    void setIPK(float i) {
        if (i >= 0.0 && i <= 4.0) {
            ipk = i;
        } else {
            cout << "Error: IPK harus 0.0 - 4.0!" << endl;
        }
    }
};

Inheritance (Pewarisan)

Inheritance: Mekanisme dimana class dapat mewarisi atribut dan method dari class lain

Base Class (Parent) ‚Üí Class yang diwarisi

Derived Class (Child) ‚Üí Class yang mewarisi

Sintaks Inheritance


// Base class
class Person {
protected:
    string nama;
    int umur;

public:
    Person(string n, int u) : nama(n), umur(u) {}
    
    void tampilkanInfo() {
        cout << "Nama: " << nama << endl;
        cout << "Umur: " << umur << endl;
    }
};

// Derived class
class Mahasiswa : public Person {
private:
    string nim;
    
public:
    Mahasiswa(string n, int u, string ni) 
        : Person(n, u), nim(ni) {}
};

Keuntungan Inheritance

‚úì Code Reusability

Tidak perlu menulis ulang kode yang sama

‚úì Extensibility

Mudah menambahkan class baru

‚úì Hierarchical Organization

Class terorganisir dalam hierarki yang logis

Urutan Constructor dan Destructor

Constructor: Base ‚Üí Derived

Destructor: Derived ‚Üí Base


class Person {
public:
    Person() { cout << "1. Constructor Person" << endl; }
    ~Person() { cout << "3. Destructor Person" << endl; }
};

class Mahasiswa : public Person {
public:
    Mahasiswa() { cout << "2. Constructor Mahasiswa" << endl; }
    ~Mahasiswa() { cout << "2. Destructor Mahasiswa" << endl; }
};

Polymorphism (Polimorfisme)

Polymorphism: "Banyak bentuk" - kemampuan menggunakan satu interface untuk berbagai tipe objek

Compile-time

Function overloading

Operator overloading

Runtime

Virtual functions

Function overriding

Function Overriding


class BangunDatar {
public:
    void hitungLuas() {
        cout << "Luas belum didefinisikan" << endl;
    }
};

class Persegi : public BangunDatar {
private:
    double sisi;
public:
    Persegi(double s) : sisi(s) {}
    
    // Override method dari base class
    void hitungLuas() {
        cout << "Luas persegi: " << sisi * sisi << endl;
    }
};

Virtual Functions

Virtual function memungkinkan runtime polymorphism

Method yang dipanggil ditentukan pada saat runtime berdasarkan tipe objek sebenarnya


class BangunDatar {
public:
    virtual double hitungLuas() {
        return 0.0;
    }
};

class Persegi : public BangunDatar {
private:
    double sisi;
public:
    double hitungLuas() override {
        return sisi * sisi;
    }
};

Runtime Polymorphism


int main() {
    BangunDatar* bentuk[3];
    
    bentuk[0] = new Persegi(5.0);
    bentuk[1] = new Lingkaran(7.0);
    bentuk[2] = new Segitiga(4.0, 6.0);
    
    // Polymorphism!
    for (int i = 0; i < 3; i++) {
        cout << "Luas: " << bentuk[i]->hitungLuas() << endl;
    }
    
    // Cleanup
    for (int i = 0; i < 3; i++) {
        delete bentuk[i];
    }
    
    return 0;
}

Virtual Destructor

Penting! Jika class memiliki virtual functions, destructor juga harus virtual


class Base {
public:
    virtual void someMethod() { }
    
    // Virtual destructor
    virtual ~Base() {
        cout << "Destructor Base" << endl;
    }
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "Destructor Derived" << endl;
    }
};

Keyword override (C++11)

override memastikan method benar-benar meng-override method base class

Compiler akan error jika signature tidak cocok


class Base {
public:
    virtual void print(int x) { }
};

class Derived : public Base {
public:
    // Benar - akan meng-override
    void print(int x) override { }
    
    // Error - signature berbeda
    // void print(double x) override { }
};

Contoh Aplikasi OOP

Sistem Perpustakaan Sederhana

Class yang dibutuhkan:

  • ItemPerpustakaan (base class) - dengan virtual methods
  • Buku (derived class) - dengan atribut pengarang, tahun
  • Majalah (derived class) - dengan atribut penerbit, edisi
  • Perpustakaan - mengelola koleksi items

Fitur: Tambah item, pinjam, kembalikan, tampilkan koleksi

Best Practices OOP

‚úì Gunakan enkapsulasi - data private, interface public

‚úì Gunakan initializer list di constructor

‚úì Buat virtual destructor untuk class dengan virtual functions

‚úì Gunakan keyword override untuk clarity

‚úì Prefer composition over inheritance (dalam beberapa kasus)

Kesalahan Umum

‚úó Mengakses private members langsung dari luar class

‚úó Lupa memanggil base class constructor

‚úó Tidak membuat destructor virtual untuk inheritance

‚úó Memory leak karena tidak delete pointer

‚úó Menggunakan inheritance hanya untuk code reuse (gunakan composition)

Kapan Menggunakan OOP?

Gunakan OOP ketika:

  • Program kompleks dan besar
  • Banyak entitas dengan atribut
  • Perlu maintainability tinggi
  • Tim development besar
  • Perlu extensibility

Prosedural cukup ketika:

  • Program kecil dan sederhana
  • Algoritma linear
  • Performance critical
  • Script utility sederhana
  • Embedded systems

Topik Lanjutan (Mata Kuliah PBO)

Di mata kuliah Pemrograman Berorientasi Objek, Anda akan mempelajari:

  • Operator overloading
  • Friend functions dan friend classes
  • Multiple inheritance
  • Abstract classes dan pure virtual functions
  • Templates dan generic programming
  • STL (Standard Template Library)
  • Design patterns (Singleton, Factory, Observer, dll)
  • SOLID principles

Ringkasan

  • OOP mengorganisir kode dalam objek (data + method)
  • Class adalah blueprint, object adalah instance
  • Constructor menginisialisasi object, destructor membersihkan
  • Enkapsulasi menyembunyikan data, akses via getter/setter
  • Inheritance memungkinkan code reuse
  • Polymorphism dengan virtual functions untuk flexibility
  • Empat pilar OOP: Encapsulation, Inheritance, Polymorphism, Abstraction
