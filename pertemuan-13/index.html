Bagian 1

C++ untuk Embedded Systems

Arduino, ESP32, dan IoT

Apa itu Embedded System?

Embedded system adalah sistem komputer yang dirancang untuk melakukan fungsi spesifik sebagai bagian dari sistem yang lebih besar.

Contoh di sekitar kita:
  • Mesin cuci, kulkas, microwave
  • Remote TV dan AC
  • Kamera digital
  • Sistem kontrol mobil
  • Medical devices

Karakteristik: Resource terbatas, real-time operation, task-specific

Mengapa C++ untuk Embedded?

C++ ideal untuk embedded programming karena:

  • Low-level control: Akses langsung ke hardware (registers, pins, memory)
  • Efisiensi: Minimal overhead, optimal memory usage
  • Object-oriented: Mengorganisir kode kompleks dengan baik
  • C compatibility: Dapat menggunakan C libraries dan drivers

Arduino: Platform Embedded Populer

Arduino adalah platform open-source untuk prototyping elektronik.

Komponen:
  • Mikrokontroler board (Uno, Nano, Mega)
  • Arduino IDE untuk programming
  • Ribuan libraries dan shields
Kegunaan:
  • Robotics
  • Home automation
  • IoT projects
  • Education

Struktur Program Arduino

Program Arduino (sketch) memiliki dua fungsi utama:

void setup() {
  // Inisialisasi
  // Dijalankan sekali saat startup
}

void loop() {
  // Program utama
  // Dijalankan berulang-ulang
}

Berbeda dengan main() yang kita kenal, Arduino abstrak away program entry point.

Program Pertama: Blink LED

Program klasik "Hello World" di embedded programming:

const int LED_PIN = 13;
const int DELAY_MS = 1000;

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_PIN, HIGH);  // Nyalakan LED
  delay(DELAY_MS);              // Tunggu 1 detik
  digitalWrite(LED_PIN, LOW);   // Matikan LED
  delay(DELAY_MS);              // Tunggu 1 detik
}

Sederhana namun mendemonstrasikan konsep fundamental I/O.

Digital I/O Concepts

Output:

Mengirim HIGH (5V) atau LOW (0V) ke pin

  • Mengontrol LED, relay, motor
pinMode(pin, OUTPUT);
digitalWrite(pin, HIGH);
Input:

Membaca HIGH atau LOW dari pin

  • Membaca tombol, switch, sensor digital
pinMode(pin, INPUT_PULLUP);
int state = digitalRead(pin);

Demo: Button Control LED

const int BUTTON_PIN = 2;
const int LED_PIN = 13;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
}

void loop() {
  int buttonState = digitalRead(BUTTON_PIN);
  
  if (buttonState == LOW) {  // Ditekan (pull-up)
    digitalWrite(LED_PIN, HIGH);
  } else {
    digitalWrite(LED_PIN, LOW);
  }
}

Input mempengaruhi output - interaksi dasar.

Analog Input

Pins analog (A0-A5) membaca voltage 0-5V menjadi nilai 0-1023.

Penggunaan:
  • Sensor suhu, cahaya, kelembaban
  • Potensiometer untuk input variable
  • Sensor jarak analog
int sensorValue = analogRead(A0);
float voltage = sensorValue * (5.0 / 1023.0);

ADC (Analog to Digital Converter) mengkonversi voltage analog menjadi nilai digital.

Demo: Sensor Suhu LM35

const int TEMP_PIN = A0;

void setup() {
  Serial.begin(9600);
}

void loop() {
  int sensorValue = analogRead(TEMP_PIN);
  float voltage = sensorValue * (5.0 / 1023.0);
  float temperatureC = voltage * 100.0;  // LM35: 10mV/¬∞C
  
  Serial.print("Temperature: ");
  Serial.print(temperatureC);
  Serial.println(" C");
  
  delay(1000);
}

Serial memungkinkan Arduino berkomunikasi dengan komputer atau devices lain.

Kegunaan:

ESP32: Next Level

ESP32 adalah mikrokontroler powerful dengan fitur:

Hardware:
  • Dual-core processor hingga 240 MHz
  • 520KB RAM
  • WiFi dan Bluetooth built-in
  • Touch sensors
Software:
  • Compatible dengan Arduino IDE
  • RTOS support
  • Powerful libraries
  • Perfect untuk IoT!

Demo: ESP32 WiFi Connection

#include <WiFi.h>

const char* ssid = "YourWiFi";
const char* password = "YourPassword";

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  
  Serial.println("\nWiFi connected!");
  Serial.print("IP: ");
  Serial.println(WiFi.localIP());
}

Dengan WiFi, ESP32 bisa terhubung ke cloud, smartphone apps, atau IoT networks.

Fast I/O Techniques

Standard cin/cout bisa lambat untuk input besar. Optimasi:

#include <iostream>
using namespace std;

int main() {
    // Optimasi I/O
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n;
    cin >> n;
    
    for(int i = 0; i < n; i++) {
        int x;
        cin >> x;
        cout << x * 2 << "\n";  // "\n" lebih cepat dari endl
    }
    
    return 0;
}

Untuk jutaan inputs, perbedaan bisa 10x lebih cepat!

STL Data Structures

Vector - dynamic array: 
vector<int> arr(n);
arr.push_back(5);
sort(arr.begin(), arr.end());
Set - unique sorted elements: 
set<int> s;
s.insert(5);  // O(log n)
if(s.count(5)) { ... }
Map - key-value pairs: 
map<string, int> freq;
freq["hello"]++;

Common Algorithms

Sorting: O(n log n) 
sort(arr.begin(), arr.end());
sort(arr.begin(), arr.end(), greater<int>());  // Descending
Binary Search: O(log n) - requires sorted array 
bool found = binary_search(arr.begin(), arr.end(), target);
auto it = lower_bound(arr.begin(), arr.end(), target);
Max/Min: 
int maxVal = *max_element(arr.begin(), arr.end());

Contoh Problem: Two Sum

Problem: Diberikan array dan target, find dua numbers yang sum = target.

Naive O(n¬≤): 
for(int i = 0; i < n; i++)
  for(int j = i+1; j < n; j++)
    if(arr[i] + arr[j] == target)
      return {i, j};
Optimal O(n): 
map<int, int> seen;
for(int i = 0; i < n; i++) {
  int comp = target - arr[i];
  if(seen.count(comp))
    return {seen[comp], i};
  seen[arr[i]] = i;
}

The Game Loop

Inti dari setiap game: infinite loop yang terus update dan render.

while (window.isOpen()) {
    // 1. Handle Input
    processEvents();
    
    // 2. Update Game State
    update(deltaTime);
    
    // 3. Render
    render();
}

Loop ini berjalan 60+ times per second (60 FPS)!

Basic SFML Program

#include <SFML/Graphics.hpp>

int main() {
    sf::RenderWindow window(sf::VideoMode(800, 600), "My Game");
    
    while (window.isOpen()) {
        sf::Event event;
        while (window.pollEvent(event)) {
            if (event.type == sf::Event::Closed)
                window.close();
        }
        
        window.clear(sf::Color::Black);
        // Draw game objects
        window.display();
    }
    
    return 0;
}

Sprite dan Graphics

Sprite = 2D image yang merepresentasikan game object.

// Load texture
sf::Texture playerTexture;
playerTexture.loadFromFile("player.png");

// Create sprite
sf::Sprite playerSprite;
playerSprite.setTexture(playerTexture);
playerSprite.setPosition(100, 100);

// In render loop
window.draw(playerSprite);

Texture adalah image data, Sprite adalah instance yang bisa di-position, scale, rotate.

Movement dengan Delta Time

Agar movement smooth dan consistent di berbagai frame rates:

sf::Clock clock;
float speed = 200.0f;  // pixels per second

while (window.isOpen()) {
    float dt = clock.restart().asSeconds();
    
    // Movement
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::W))
        playerSprite.move(0, -speed * dt);
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::S))
        playerSprite.move(0, speed * dt);
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::A))
        playerSprite.move(-speed * dt, 0);
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::D))
        playerSprite.move(speed * dt, 0);
}

Delta time = waktu yang elapsed sejak frame terakhir.

Collision Detection

Detect ketika objects bersentuhan:

Bounding box: Rectangular area yang contain sprite.

Advanced: Circle collision, pixel-perfect collision, polygon collision.

Cross-Platform Development

Write Once, Compile Anywhere

Platform Challenges

Develop untuk multiple OS (Windows, macOS, Linux) memiliki challenges:

  • Kompiler berbeda: GCC, Clang, MSVC
  • System APIs berbeda: Windows API vs POSIX
  • Path separators: Windows \ vs Unix /
  • Line endings: Windows \r\n vs Unix \n
  • Case sensitivity: Linux case-sensitive, Windows tidak

Compiler Perbedaan

Compiler Platform Karakteristik
GCC Linux (default) Open-source, excellent C++ support
Clang macOS (default) Part of LLVM, clear error messages
MSVC Windows Visual Studio, Windows-optimized

Preprocessor untuk Platform Detection

#ifdef _WIN32
    // Windows-specific code
    #include <windows.h>
#elif __linux__
    // Linux-specific code
    #include <unistd.h>
#elif __APPLE__
    // macOS-specific code
    #include <unistd.h>
#endif

Conditional compilation memungkinkan different code untuk different platforms.

Example: Cross-Platform Sleep

#include <iostream>

void sleep_ms(int milliseconds) {
    #ifdef _WIN32
        #include <windows.h>
        Sleep(milliseconds);
    #else
        #include <unistd.h>
        usleep(milliseconds * 1000);
    #endif
}

int main() {
    std::cout << "Waiting 2 seconds...\n";
    sleep_ms(2000);
    std::cout << "Done!\n";
    return 0;
}

Same interface, different implementation per platform.

Build Systems: Makefile

Traditional build system untuk Unix:

CXX = g++
CXXFLAGS = -std=c++17 -Wall

myprogram: main.o utils.o
	$(CXX) -o myprogram main.o utils.o

main.o: main.cpp
	$(CXX) $(CXXFLAGS) -c main.cpp

clean:
	rm -f *.o myprogram

Compile hanya files yang berubah, efficient untuk large projects.

CMake: Modern Build System

CMake generate platform-specific build files:

cmake_minimum_required(VERSION 3.10)
project(MyGame VERSION 1.0)

set(CMAKE_CXX_STANDARD 17)

add_executable(MyGame 
    src/main.cpp
    src/game.cpp
    src/player.cpp
)

target_include_directories(MyGame PRIVATE include)
Workflow: 
mkdir build && cd build
cmake ..
cmake --build .

CMake Keuntungan

  • ‚úÖ Cross-platform: Same CMakeLists.txt untuk semua OS
  • ‚úÖ IDE integration: Generate Visual Studio, Xcode projects
  • ‚úÖ Dependency management: Find dan link libraries dengan mudah
  • ‚úÖ Out-of-source builds: Build files terpisah dari source
  • ‚úÖ Widely adopted: Industry standard

Best Practices Cross-Platform

  • ‚úÖ Use standard C++ sebisa mungkin - hindari compiler-specific extensions
  • ‚úÖ Test di semua target platforms - jangan assume behavior sama
  • ‚úÖ Abstract platform-specific code - create wrapper functions/classes
  • ‚úÖ Use portable libraries - Boost, SDL, SFML, Qt
  • ‚úÖ Document platform differences - clear build instructions

Debugging Tools

Finding and Fixing Bugs Efficiently

Debugging: Essential Skill

Debugging adalah process menemukan dan fixing bugs dalam code.

Tools vary by platform:
  • Linux: GDB (GNU Debugger)
  • macOS: LLDB
  • Windows: Visual Studio Debugger
  • Cross-platform IDEs: CLion, VSCode

Bugs are inevitable, good debugging saves time!

GDB (GNU Debugger)

Command-line debugger untuk Linux/Unix:

# Compile dengan debug symbols
g++ -g -o program main.cpp

# Run GDB
gdb ./program

# Basic commands:
(gdb) break main          # Set breakpoint
(gdb) run                 # Run program
(gdb) next                # Step over
(gdb) step                # Step into
(gdb) print variable      # Print value
(gdb) backtrace          # Show call stack
(gdb) continue           # Continue execution

LLDB untuk macOS

Similar to GDB dengan syntax sedikit berbeda:

lldb ./program

(lldb) breakpoint set --name main
(lldb) run
(lldb) next
(lldb) step
(lldb) print variable
(lldb) bt
(lldb) continue

LLDB default debugger di Xcode.

Visual Studio Debugger

GUI-based debugger yang powerful:

Features:
  • Click untuk set breakpoints
  • Hover untuk inspect variables
  • Watch window
  • Call stack visualization
  • Memory viewer
Shortcuts:
  • F5: Start debugging
  • F9: Toggle breakpoint
  • F10: Step over
  • F11: Step into

Debugging dalam IDE

Modern IDEs provide integrated debugging:

CLion:
  • Visual breakpoints
  • Inline variable values
  • CMake integration
VSCode:
  • IntelliSense
  • Integrated terminal
  • Multi-platform support

Debugging Arduino

Embedded debugging lebih challenging:

Serial debugging: 
Serial.begin(9600);
Serial.print("Debug: variable = ");
Serial.println(variable);
LED indicators: 
// Blink different patterns untuk different states
if (errorState) {
    blinkFast();  // Error indicator

Advanced: JTAG debuggers untuk hardware-level debugging.

Debugging Best Practices

  • ‚úÖ Use version control (Git) - Bisa rollback ke working version
  • ‚úÖ Write testable code - Small functions, clear responsibilities
  • ‚úÖ Add assertions - assert(index >= 0 && index < size);
  • ‚úÖ Use debugger, not just print - More efficient untuk complex bugs
  • ‚úÖ Reproduce bugs consistently - Easier to fix

Common Bug Categories

Category Description
Syntax errors Compiler catches these
Logic errors Program runs but wrong output
Runtime errors Crashes, segfaults, exceptions
Memory errors Leaks, dangling pointers
Off-by-one errors Loop boundaries

Setiap category memerlukan approach berbeda untuk debug!
